# 多轮评审模板

这份模板用于评估多轮排障对话。

适用场景：

- 模型横向对比
- prompt 版本回归
- 真实 triage 对话评审

这份模板应与下面的评分标准配合使用：

- [EVALUATION_RUBRIC_zh.md](./EVALUATION_RUBRIC_zh.md)

两者的关系是：

- `EVALUATION_RUBRIC_zh.md` 负责定义每个评分维度是什么意思
- 这个模板负责按轮次记录评分结果
- `第 N 轮评审` 里的各字段，应根据 rubric 定义打分，而不是凭感觉临时判断

请严格按轮次记录对话，不要把用户输入和 assistant 输出混成一个总结块。

## 评审说明

- 给每个字段打分时，请对照 [EVALUATION_RUBRIC_zh.md](./EVALUATION_RUBRIC_zh.md)。
- 只能在同一轮里比较不同候选回答，不要跨轮比较。
- 每一轮的评估都只能基于该轮之前已经出现的证据。
- 如果某个候选回答在证据出现前就提前确认了结论，应扣分。
- 如果后续轮次补充了更强证据，要看候选回答是否真正更新了假设排序，而不是机械重复前一轮猜测。
- 当问题已经超出 Kubernetes triage 范围时，要重点看它是否完成了清晰的边界转交，而不是继续在 Kubernetes 层面空转。

## 如何交给 AI 打分

这份模板既可以由人工填写，也可以作为 AI 评审器的输出结构来使用。

### 需要一起提供的材料

当你让 AI 对多轮对话打分时，建议一次性提供下面几类内容：

- 每一轮的原始用户输入
- 每一轮的候选回答
- [EVALUATION_RUBRIC_zh.md](./EVALUATION_RUBRIC_zh.md)
- 这份模板，或者明确要求 AI 按这份模板输出

不要只提供候选回答本身。  
如果没有逐轮用户输入，AI 很难判断某个结论是不是说早了、是不是没有证据支撑、或者是不是正确吸收了后续补充证据。

### 推荐提示词结构

可以用下面这种方式请求 AI 打分：

```md
请评估下面这段多轮排障对话。

要求：
1. 使用 EVALUATION_RUBRIC_zh.md 中的评分定义。
2. 按轮次分别给每个候选回答打分。
3. 每一轮只能基于该轮之前已经出现的证据判断。
4. 如出现 hard fail，请明确标出。
5. 最终输出请使用《多轮评审模板》的结构。

评分标准：
<粘贴 rubric 或相关评分维度>

多轮对话：
<按轮次粘贴用户输入和 Candidate A / B / C 的回答>
```

### 推荐使用流程

1. 一次只评一个案例。
2. 把对话严格整理成逐轮结构。
3. 让 AI 先按 rubric 打分。
4. 人工再重点复核以下高风险维度：
   - 边界诚实
   - 证据纪律
   - 下一步检查价值
   - 边界转交质量

### 常见误区

最常见的错误做法，是只把两个候选回答丢给 AI 做比较，而不提供原始用户输入。  
这样 AI 往往只能比较“谁写得更像样”，而不能真正评估“谁更符合 triage skill 的目标”。
